
# 챕터 0. 오디오 데이터 요약

## 핵심 요약

오디오는 원래 연속적인 아날로그 신호이지만, 컴퓨터에서 다루기 위해서는 이산적인 숫자 배열로 바꿔야 한다. 이 과정에서 중요한 개념은 샘플링 속도, 진폭, 비트뎁스, 파형, 주파수 스펙트럼, 스펙트로그램, 멜 스펙트로그램이다.

머신러닝에서 오디오는 보통 다음 둘 중 하나의 형태로 다룬다.

- 시간에 따른 샘플 값 배열인 파형
- 시간과 주파수 정보를 함께 담은 스펙트로그램 또는 멜 스펙트로그램

## 1. 오디오의 디지털 표현

소리는 본질적으로 연속적인 음파다. 하지만 컴퓨터는 무한히 이어지는 값을 그대로 저장하거나 계산할 수 없기 때문에, 일정한 간격으로 값을 뽑아 숫자 배열로 저장한다.

일반적인 오디오 파일 형식은 다음과 같다.

- `.wav`: 비압축 또는 낮은 압축의 파형 오디오 형식
- `.flac`: 무손실 압축 오디오 형식
- `.mp3`: 손실 압축 오디오 형식

이 형식들은 주로 오디오 신호를 압축하고 저장하는 방식이 다르다. 모델 학습에서는 파일 형식 자체보다 실제로 로딩된 오디오 배열, 샘플링 속도, 채널 수, 길이, 정규화 방식이 더 중요하다.

## 2. 샘플링과 샘플링 속도

샘플링은 연속적인 소리 신호를 일정한 시간 간격으로 측정하는 과정이다.

샘플링 속도는 1초 동안 몇 개의 샘플을 뽑는지를 뜻하며, 단위는 Hz다.

예시:

- 16,000 Hz: 음성 모델에서 자주 사용
- 44,100 Hz: CD 품질 오디오
- 192,000 Hz: 고해상도 오디오

음성 인식에서는 보통 16 kHz가 많이 쓰인다. 사람 음성의 주요 정보가 대체로 8 kHz 이하에 있기 때문이다. 샘플링 속도가 16 kHz이면 나이퀴스트 한계에 따라 최대 8 kHz 정도의 주파수를 표현할 수 있다.

중요한 점은 데이터셋과 모델의 샘플링 속도를 맞춰야 한다는 것이다. 예를 들어 사전학습 모델이 16 kHz 오디오로 학습되었다면, fine-tuning 데이터도 16 kHz로 맞추는 것이 좋다.

샘플링 속도가 다르면 같은 길이의 오디오라도 입력 배열 길이가 달라진다.

```text
5초 오디오, 16 kHz → 80,000개 샘플
5초 오디오, 8 kHz  → 40,000개 샘플
```

이 차이는 트랜스포머 모델이 보는 시퀀스 길이와 시간 해상도에 직접 영향을 준다.

## 3. 리샘플링

리샘플링은 서로 다른 샘플링 속도를 하나의 기준으로 맞추는 전처리 작업이다.

예를 들어 여러 음성 파일이 각각 8 kHz, 16 kHz, 44.1 kHz로 되어 있다면, 학습 전에 모두 16 kHz로 변환할 수 있다.

STT 학습에서는 보통 다음을 확인해야 한다.

- 모든 오디오의 샘플링 속도가 같은가?
- 모델이 기대하는 샘플링 속도와 같은가?
- 리샘플링 후 음질이 과도하게 손상되지 않았는가?

## 4. 진폭과 비트뎁스

진폭은 특정 시점의 소리 크기를 나타낸다. 디지털 오디오에서는 각 샘플 값이 그 순간의 진폭을 표현한다.

비트뎁스는 진폭 값을 얼마나 정밀하게 저장할지를 결정한다.

대표적인 비트뎁스:

- 16비트: 일반적인 오디오에서 많이 사용
- 24비트: 더 정밀한 오디오 표현
- 32비트 float: 머신러닝 처리에서 자주 사용되는 부동소수점 표현

비트뎁스가 높을수록 진폭을 더 세밀하게 표현할 수 있지만, 음성 모델 학습에서 항상 높은 비트뎁스가 필요한 것은 아니다. 머신러닝 모델은 보통 부동소수점 텐서를 입력으로 받기 때문에, 학습 전 오디오를 float 형태로 변환하는 과정이 들어간다.

## 5. 데시벨과 디지털 오디오

사람의 청각은 선형적이지 않고 로그 스케일에 가깝다. 그래서 소리 크기를 표현할 때 데시벨(dB)을 자주 사용한다.

현실 세계의 소리 크기와 디지털 오디오의 dB 기준은 다르다.

- 현실 소리: 조용한 소리를 기준으로 dB가 커질수록 큰 소리
- 디지털 오디오: 최대 진폭을 0 dBFS로 보고, 그보다 작은 값은 음수 dB로 표현

오디오 분석이나 시각화에서 dB 스케일을 쓰면 작은 신호 차이를 더 보기 좋게 만들 수 있다.

## 6. 파형

파형은 시간에 따른 오디오 샘플 값을 그래프로 나타낸 것이다.

파형 그래프:

- X축: 시간
- Y축: 진폭

파형을 보면 다음을 확인할 수 있다.

- 소리가 언제 발생했는지
- 전체적인 음량이 어느 정도인지
- 묵음 구간이 있는지
- 노이즈나 이상한 튐이 있는지
- 전처리 결과가 이상하지 않은지

예를 들어 `librosa`를 사용하면 오디오 파일을 배열과 샘플링 속도로 불러오고, 파형을 시각화할 수 있다.

```python
import librosa
import librosa.display
import matplotlib.pyplot as plt

array, sampling_rate = librosa.load(librosa.ex("trumpet"))

plt.figure().set_figwidth(12)
librosa.display.waveshow(array, sr=sampling_rate)
plt.show()
```

## 7. 주파수 스펙트럼

파형이 시간 영역 표현이라면, 주파수 스펙트럼은 주파수 영역 표현이다.

주파수 스펙트럼은 오디오 신호 안에 어떤 주파수 성분이 얼마나 강하게 들어 있는지 보여준다. 이를 계산할 때 이산 푸리에 변환(DFT)을 사용한다.

스펙트럼 그래프:

- X축: 주파수
- Y축: 진폭 또는 파워

일반적으로 주파수 축은 로그 스케일로 보는 경우가 많다. 음악 소리에서는 기본 주파수와 고조파가 피크로 나타날 수 있다.

머신러닝에서는 위상 정보보다 진폭 또는 파워 정보가 더 자주 사용된다.

간단한 예시는 다음과 같다.

```python
import numpy as np
import librosa
import matplotlib.pyplot as plt

dft_input = array[:4096]
window = np.hanning(len(dft_input))
windowed_input = dft_input * window
dft = np.fft.rfft(windowed_input)

amplitude = np.abs(dft)
amplitude_db = librosa.amplitude_to_db(amplitude, ref=np.max)
frequency = librosa.fft_frequencies(sr=sampling_rate, n_fft=len(dft_input))

plt.figure().set_figwidth(12)
plt.plot(frequency, amplitude_db)
plt.xlabel("Frequency (Hz)")
plt.ylabel("Amplitude (dB)")
plt.xscale("log")
plt.show()
```

## 8. 스펙트로그램

스펙트럼은 특정 구간의 주파수 성분을 보여준다. 하지만 실제 오디오는 시간에 따라 주파수 성분이 계속 바뀐다.

스펙트로그램은 짧은 시간 구간마다 주파수 스펙트럼을 계산하고, 그것을 시간축으로 쌓은 표현이다.

스펙트로그램:

- X축: 시간
- Y축: 주파수
- 색상: 진폭 또는 파워

스펙트로그램을 만들 때는 보통 STFT를 사용한다. STFT는 오디오를 짧은 구간으로 나눈 뒤 각 구간에 푸리에 변환을 적용하는 방식이다.

스펙트로그램은 오디오 분석에서 매우 중요하다. 음성에서는 모음이나 발음 특성이 주파수 패턴으로 드러나고, 음악에서는 악기나 보컬의 주파수 구성이 보인다.

예시:

```python
import numpy as np
import librosa
import librosa.display
import matplotlib.pyplot as plt

D = librosa.stft(array)
S_db = librosa.amplitude_to_db(np.abs(D), ref=np.max)

plt.figure().set_figwidth(12)
librosa.display.specshow(S_db, x_axis="time", y_axis="hz")
plt.colorbar()
plt.show()
```

## 9. 스펙트로그램과 파형의 관계

파형과 스펙트로그램은 같은 오디오 데이터를 다른 관점에서 본 것이다.

- 파형: 시간에 따른 진폭 변화
- 스펙트로그램: 시간에 따른 주파수 성분 변화

일반 스펙트로그램은 원래 파형으로 어느 정도 복원할 수 있다. 하지만 이때는 진폭 정보뿐 아니라 위상 정보도 필요하다. 머신러닝 모델이 진폭만 예측하는 경우에는 vocoder나 Griffin-Lim 같은 방법으로 파형을 재구성해야 할 수 있다.

## 10. 멜 스펙트로그램

멜 스펙트로그램은 음성 처리와 머신러닝에서 자주 쓰는 스펙트로그램 변형이다.

일반 스펙트로그램은 주파수 축이 Hz 기준의 선형 스케일이다. 반면 멜 스펙트로그램은 사람의 청각 특성을 반영한 멜 스케일을 사용한다.

사람은 고주파보다 저주파 변화에 더 민감하다. 멜 스케일은 이런 비선형적인 청각 특성을 근사한다.

멜 스펙트로그램을 만드는 흐름:

```text
오디오 파형
→ STFT
→ 주파수 스펙트럼
→ mel filterbank 적용
→ 멜 스펙트로그램
→ 로그 또는 dB 변환
```

예시:

```python
import numpy as np
import librosa
import librosa.display
import matplotlib.pyplot as plt

S = librosa.feature.melspectrogram(
    y=array,
    sr=sampling_rate,
    n_mels=128,
    fmax=8000
)
S_dB = librosa.power_to_db(S, ref=np.max)

plt.figure().set_figwidth(12)
librosa.display.specshow(
    S_dB,
    x_axis="time",
    y_axis="mel",
    sr=sampling_rate,
    fmax=8000
)
plt.colorbar()
plt.show()
```

## 11. 멜 스펙트로그램에서 주의할 점

멜 스펙트로그램은 구현 방식이 항상 같지 않다.

확인해야 할 항목:

- 사용하는 멜 스케일이 `htk`인지 `slaney`인지
- 진폭 스펙트로그램을 쓰는지 파워 스펙트로그램을 쓰는지
- 로그 변환 방식이 실제 dB 변환인지 단순 로그인지
- `n_mels` 값이 몇 개인지
- `fmax` 값이 모델 설정과 맞는지

사전학습 모델을 fine-tuning할 때는 모델이 학습 당시 사용한 오디오 전처리 방식과 최대한 맞춰야 한다.

## 12. STT 학습 관점에서 중요한 부분

STT를 학습할 때 특히 중요한 것은 다음이다.

- 오디오 샘플링 속도를 모델 요구사항에 맞추기
- 오디오를 float 배열 또는 텐서로 변환하기
- 너무 긴 오디오를 적절히 자르거나 패딩하기
- 묵음, 잡음, 비정상 파일을 확인하기
- 전처리 후 파형과 스펙트로그램을 시각화해서 품질 확인하기
- 모델이 요구하는 feature extractor 설정을 그대로 따르기

Whisper 같은 모델을 사용할 때는 보통 직접 멜 스펙트로그램을 손으로 만들기보다, `AutoProcessor`나 모델 전용 processor가 필요한 전처리를 수행하게 하는 것이 안전하다.

## 13. 한 줄 정리

오디오 데이터는 단순한 소리 파일이 아니라 시간에 따른 숫자 시퀀스이며, 샘플링 속도와 주파수 표현 방식을 제대로 맞추는 것이 STT 모델 학습의 기본이다.
