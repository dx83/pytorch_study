# 오디오 데이터셋 불러오기 및 탐색하기 요약

출처: https://huggingface.co/learn/audio-course/ko/chapter1/load_and_explore

## 핵심 내용

Hugging Face의 `Datasets` 라이브러리는 오디오를 포함한 다양한 형태의 머신러닝 데이터셋을 쉽게 다운로드하고 준비할 수 있게 해준다. 특히 오디오 데이터셋을 다룰 때 필요한 디코딩, 샘플링 레이트 확인, 배열 접근 같은 기능을 제공한다.

이 장에서는 `MINDS-14` 오디오 데이터셋을 예제로 사용해 데이터셋을 불러오고, 각 샘플의 구조를 살펴보고, 필요 없는 컬럼을 제거한 뒤, 오디오를 직접 듣거나 파형으로 시각화하는 과정을 다룬다.

## 준비

오디오 데이터셋을 다루기 위해 다음 패키지를 설치한다.

```bash
pip install datasets[audio] soundfile librosa torchcodec
```

## 데이터셋 불러오기

`Datasets` 라이브러리의 가장 중요한 기능 중 하나는 `load_dataset()` 함수 한 줄로 Hugging Face Hub의 데이터셋을 다운로드하고 준비할 수 있다는 점이다.

예제에서는 여러 언어와 방언으로 녹음된 인터넷뱅킹 관련 발화를 담은 `MINDS-14` 데이터셋을 사용한다. 데이터셋 식별자는 `PolyAI/minds14`이며, 호주 영어 부분인 `en-AU`의 학습 데이터만 불러온다.

```python
from datasets import load_dataset

minds = load_dataset("PolyAI/minds14", name="en-AU", split="train")
minds
```

불러온 데이터셋은 654개의 오디오 파일로 구성되어 있으며, 주요 컬럼은 다음과 같다.

- `path`: 오디오 파일 경로
- `audio`: 디코딩된 오디오 데이터
- `transcription`: 원문 전사
- `english_transcription`: 영어 전사
- `intent_class`: 발화 의도 레이블
- `lang_id`: 언어 ID

## 오디오 샘플 구조

첫 번째 샘플을 확인하면 각 예제가 어떤 정보를 담고 있는지 볼 수 있다.

```python
example = minds[0]
example
```

`audio` 컬럼은 다시 다음 세 가지 정보를 포함한다.

- `path`: 실제 오디오 파일 경로
- `array`: 오디오를 디코딩한 1차원 NumPy 배열
- `sampling_rate`: 샘플링 레이트

예제 데이터의 샘플링 레이트는 8,000 Hz이다. `array`는 모델 입력이나 시각화에 사용할 수 있는 실제 오디오 신호 값이다.

## 레이블 확인

`intent_class`는 숫자 형태의 레이블이다. 이 값을 사람이 읽기 쉬운 문자열로 바꾸려면 feature의 `int2str()` 메서드를 사용한다.

```python
id2label = minds.features["intent_class"].int2str
id2label(example["intent_class"])
```

예를 들어 첫 번째 예제의 레이블은 `"pay_bill"`로 변환된다. 전사 텍스트를 보면 사용자가 카드로 전기요금을 내고 싶다고 말하고 있으므로, 이 레이블과 내용이 잘 대응된다.

## 불필요한 컬럼 제거

특정 작업에는 모든 컬럼이 필요하지 않을 수 있다. 예를 들어 `en-AU` 데이터만 사용하는 경우 `lang_id`는 모든 샘플에서 동일하므로 유용성이 낮다. 또한 이 부분집합에서는 `english_transcription`이 `transcription`과 대부분 같을 수 있다.

이런 컬럼은 `remove_columns()`로 제거할 수 있다.

```python
columns_to_remove = ["lang_id", "english_transcription"]
minds = minds.remove_columns(columns_to_remove)
minds
```

제거 후 데이터셋은 다음 컬럼만 남는다.

- `path`
- `audio`
- `transcription`
- `intent_class`

## 오디오 들어보기

데이터셋을 눈으로만 확인하는 것보다 실제 오디오를 들어보는 것이 중요하다. 예제에서는 `Gradio`의 `Blocks`와 `Audio` 컴포넌트를 사용해 무작위 샘플 몇 개를 재생한다.

```python
import gradio as gr

def generate_audio():
    example = minds.shuffle()[0]
    audio = example["audio"]
    return (
        audio["sampling_rate"],
        audio["array"],
    ), id2label(example["intent_class"])

with gr.Blocks() as demo:
    with gr.Column():
        for _ in range(4):
            audio, label = generate_audio()
            output = gr.Audio(audio, label=label)

demo.launch(debug=True)
```

이 코드는 데이터셋을 섞은 뒤 하나의 샘플을 골라 오디오 신호와 레이블을 함께 보여준다.

## 파형 시각화

오디오 데이터를 더 잘 이해하려면 파형을 시각화할 수 있다. 첫 번째 예제의 오디오 배열과 샘플링 레이트를 가져와 `librosa`와 `matplotlib`으로 파형을 그린다.

```python
import librosa
import matplotlib.pyplot as plt
import librosa.display

array = example["audio"]["array"]
sampling_rate = example["audio"]["sampling_rate"]

plt.figure().set_figwidth(12)
librosa.display.waveshow(array, sr=sampling_rate)
```

파형은 시간에 따른 오디오 신호의 진폭 변화를 보여준다. 이를 통해 녹음 길이, 음성 구간, 무음 구간 등을 대략적으로 파악할 수 있다.

## 정리

이 장의 전체 흐름은 다음과 같다.

1. `datasets[audio]`와 관련 오디오 패키지를 설치한다.
2. `load_dataset()`으로 Hugging Face Hub의 오디오 데이터셋을 불러온다.
3. 데이터셋의 컬럼과 샘플 구조를 확인한다.
4. `audio` 컬럼의 `path`, `array`, `sampling_rate`를 이해한다.
5. 숫자 레이블을 `int2str()`로 문자열 레이블로 변환한다.
6. 필요 없는 컬럼은 `remove_columns()`로 제거한다.
7. Gradio로 샘플 오디오를 들어본다.
8. `librosa`로 오디오 파형을 시각화한다.

마지막으로, MINDS-14의 다른 언어나 방언도 직접 불러와 들어보고 시각화해보면 전체 데이터셋의 특성을 더 잘 이해할 수 있다.
